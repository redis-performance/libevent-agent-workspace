# EXP-002 — 2026-06-01 — Skip timerfd_settime on NONBLOCK path

## Status: REJECTED

---

## Selection Phase

**Tier 5a (epoll fast path)**: skip redundant `timerfd_settime` syscall when timeout is zero.

**Hypothesis**: With `EVLOOP_NONBLOCK`, every `epoll_dispatch` call performs `timerfd_settime(fd,
0, {0,0}, NULL)` to disarm a timer, but `epoll_wait(timeout=0)` returns immediately regardless
of timerfd state — the disarm call is wasted. Eliminating it (~101 calls per cascade `run_once`)
should reduce cascade_bench by ≥2%.

---

## Implementation Phase

One-line restructure in `epoll.c` `#ifdef USING_TIMERFD` block: when `tv != NULL &&
tv->tv_sec == 0 && tv->tv_usec == 0`, set `timeout = 0` and skip `timerfd_settime`.

**Critical discovery (post-build)**: `USING_TIMERFD` is conditionally defined at line 81-92 of
`epoll.c` only when `!defined(EVENT__HAVE_EPOLL_PWAIT2)`. This system has
`EVENT__HAVE_EPOLL_PWAIT2 = 1`, so `USING_TIMERFD` is **not defined** and the entire
`#ifdef USING_TIMERFD` block is dead code. The change compiled as a **no-op**.

The hot path on this system uses `epoll_pwait2` with `struct timespec ts = {tv_sec,
tv_usec*1000}`. No `timerfd_settime` is involved at all.

## Decision

**Status**: REJECT  
**Reason**: Technique targets the `USING_TIMERFD` path, which is compiled out when
`epoll_pwait2` is available (mutually exclusive by design). Change had no effect on the binary.
Observed 16% cascade_bench regression is pure machine noise (last rep confirms normal baseline).

Added to Known Non-Starters.

---

## Lessons

- `USING_TIMERFD` and `epoll_pwait2` are mutually exclusive — always check which code path is
  compiled before targeting it.
- On this system, the hot dispatch path is `epoll_pwait2`, NOT timerfd. Future timeout-related
  optimizations should target the `epoll_pwait2` path (ts computation, `TIMEVAL_TO_TIMESPEC`).
- Benchmark noise: this machine can swing 15-20% between reps; always check raw sample
  distribution (not just median) before attributing a result to code change.
